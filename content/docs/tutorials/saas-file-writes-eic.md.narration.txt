# SaaS File Writes via EC2 Instance Connect — Narration Script

Tone: calm, technical. Audience: developers evaluating Molecule AI SaaS or debugging the Files API.
Target screencast length: ~90 seconds.

---

**[0–10 s] Opening**

"On SaaS workspaces, Molecule AI doesn't run your agent in a Docker container — it runs it on a dedicated EC2. That means the Files API had a gap: it knew how to copy files into Docker, but not how to write to an EC2 instance. EC2 Instance Connect closes that gap, using the same secure SSH channel as the Terminal tab."

**[10–25 s] Routing diagram**

"Here's the decision tree every write call goes through. First, does the workspace have an EC2 instance ID? If yes — we're on SaaS, so we go through Instance Connect. If no, we try the running Docker container. And if that's offline, an ephemeral container. For most self-hosted deployments, only the Docker path is ever exercised."

**[25–45 s] Single-file write**

"Let's write a single file. The agent prompt. A PUT to the Files API with a JSON body. The SaaS path kicks in. Molecule generates an ephemeral SSH key pair, pushes the public key to AWS for a sixty-second window, opens a TLS tunnel, and runs the install command — install with a D flag, which creates any missing parent directories atomically. File lands at the hermes runtime path: home-slash-ubuntu-slash-dot-hermes. The key pair is wiped immediately when the function returns."

**[45–60 s] Bulk replace**

"For multiple files at once, use the bulk replace endpoint. The body is a map of relative paths to content. The response tells you how many files were written and confirms the EC2 SSH source. Each file gets its own tunnel — so for ten files, expect around thirty seconds total. A tar-based follow-up is tracked as a future improvement."

**[60–75 s] Security properties**

"Let's talk security. Path traversal is blocked at the API boundary — the relative path is cleaned and checked before any handler runs. No shell injection surface: the remote command uses install, not a shell invocation, and the absolute path is built from a closed map plus clean, so there is no user-controlled string that reaches the shell. The SSH key lives for thirty seconds maximum. And AWS drops the authorized key after sixty seconds even if you do not explicitly revoke it."

**[75–90 s] Error cases and source**

"If you hit a five hundred on a SaaS workspace with no instance ID, that workspace record is misconfigured — open a support ticket. If you see a path traversal blocked error, the client sent a relative path with dot-dot. Fix it on the client side. All of this is in PR seventeen-oh-two on the molecule-core repository — linked in the description."
